# pTODOlist - Design System

## 1. 공통 UI 컴포넌트

### CheckboxTile
- 체크박스 + 제목 + 카테고리 색상 도트
- 스와이프: 왼쪽→삭제, 오른쪽→수정
- 완료 시: 취소선 + 투명도 감소 + 하단 이동

### CategoryChip
- 작은 둥근 태그 (카테고리 색상 배경 + 이름)
- 사용: 폼의 카테고리 선택, 필터

### ProgressRing
- 원형 프로그레스 인디케이터
- 중앙에 퍼센트 텍스트
- 색상: 0~49% Warning, 50~99% Primary, 100% Success

### SectionHeader
- 섹션 제목 + 우측 액션 버튼 (선택적)
- 예: "오늘의 루틴" + "편집" 버튼

### EmptyState
- 중앙 아이콘 + 설명 텍스트 + CTA 버튼
- 예: "아직 루틴이 없어요" + "루틴 추가하기"

### FAB (Floating Action Button)
- + 아이콘, 탭 시 바텀시트 열림
- 바텀시트: "루틴 추가" / "할 일 추가" 선택

---

## 2. 화면 패턴

### 리스트 화면
- 상단: 화면 제목 (AppBar)
- 중앙: 스크롤 가능한 리스트
- 하단: FAB 또는 바텀 네비게이션

### 폼 화면 (모달/바텀시트)
- 상단: 제목 + 닫기 버튼
- 중앙: 입력 필드들
- 하단: 저장 버튼

### 통계 화면
- 상단: 기간 선택 탭 (일/주/월)
- 중앙: 차트 영역
- 하단: 카테고리별 분석 리스트

## 3. 인터랙션 패턴

### 체크박스 토글
1. 탭 → 체크 애니메이션 (0.2초)
2. 프로그레스 링 업데이트 (0.3초)
3. 완료 시 항목이 하단으로 슬라이드 (0.3초)

### 스와이프 삭제
1. 왼쪽 스와이프 → 빨간 배경 + 삭제 아이콘 표시
2. 임계값 넘기면 삭제 확인 다이얼로그
3. 확인 → 항목 사라짐 애니메이션

### 항목 추가
1. FAB 탭 → 바텀시트 (루틴/할 일 선택)
2. 선택 → 폼 바텀시트 슬라이드 업
3. 저장 → 리스트에 항목 추가 애니메이션
