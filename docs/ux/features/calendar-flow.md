# 캘린더 뷰 (F13) - User Flow

## 진입 경로

- 하단 네비게이션 바 2번째 탭 "달력" 아이콘 탭
- 탭 순서: 오늘 | **달력** | 통계 | 설정

---

## 화면 구성

### 메인 화면: 월별 캘린더

```
[AppBar: "2026년 3월"]
[스트릭 배너: 🔥 7일 연속 달성 중]
[월~일 요일 헤더]
[날짜 그리드: 색상 강도로 달성률 표시]
[하단 네비게이션]
```

---

## Flow 1: 월별 캘린더 조회

1. 사용자가 하단 "달력" 탭을 탭한다
2. 현재 월의 캘린더가 표시된다
3. 각 날짜 셀은 해당 일의 달성률에 따라 색상 강도가 다르다
4. 오늘 날짜는 테두리로 강조된다
5. 미래 날짜는 비활성(회색) 상태로 표시된다

### 달성률 → 색상 매핑

```
  0%  →  빈 셀 (surfaceContainerHighest)
 1~25% →  매우 연한 primary
26~50% →  연한 primary (primaryContainer)
51~75% →  primary 60% opacity
76~100% → primary 100%
```

### 데이터 흐름

```
달력 탭 진입
  ↓
현재 월의 DailyRecord 조회 (getRecordsInRange)
  ↓
날짜별 completionRate 계산
  ↓
색상 강도 매핑 → 그리드 렌더링
  ↓
StreakCalculator로 현재 스트릭 계산 → 배너 표시
```

---

## Flow 2: 월 이동 네비게이션

1. AppBar의 좌/우 화살표를 탭한다
2. 이전/다음 월의 캘린더로 전환된다
3. 해당 월의 DailyRecord를 조회하여 렌더링한다
4. 월/년 헤더 텍스트가 업데이트된다

```
[<] 2026년 3월 [>]
         ↓
  좌 화살표 탭
         ↓
[<] 2026년 2월 [>]
```

### 제한 사항
- 미래 월로는 이동 가능하나 데이터가 없으면 빈 캘린더 표시
- 과거 월은 데이터 보관 기간 내에서만 유의미 (Settings의 보관 기간 설정에 따름)

---

## Flow 3: 일별 상세 조회

1. 캘린더의 날짜 셀을 탭한다
2. 바텀시트가 올라온다
3. 해당 날짜의 달성 요약이 표시된다:
   - 날짜 (예: "3월 15일 토요일")
   - 달성률 (예: "5/7 완료 (71%)")
4. 루틴 섹션: 해당 일의 루틴 목록 + 완료/미완료 상태
5. 할 일 섹션: 해당 일의 할 일 목록 + 완료/미완료 상태
6. 모든 항목은 **읽기 전용** (과거 데이터 수정 불가)

```
날짜 셀 탭
  ↓
DailyRecord에서 해당 날짜 조회
  ↓
routineCompletions 맵 → 루틴 이름 매칭
  ↓
AdditionalTask에서 해당 날짜 할 일 조회
  ↓
바텀시트 렌더링 (읽기 전용)
```

### 오늘 날짜 탭 시
- 동일한 바텀시트 표시 (읽기 전용)
- 수정은 홈 탭에서만 가능

---

## Flow 4: 스트릭 배너

1. 캘린더 화면 진입 시 현재 스트릭을 계산한다
2. 스트릭 배너에 연속 달성일 수를 표시한다
3. 스트릭이 0이면 배너를 숨기거나 "아직 스트릭이 없어요" 표시

```
StreakCalculator.calculateOverallStreak(records)
  ↓
streak > 0 → "🔥 {N}일 연속 달성 중"
streak == 0 → 배너 숨김 또는 격려 메시지
```

---

## Edge Cases

| 상황 | 처리 |
|---|---|
| 데이터가 없는 월 | 빈 캘린더 표시 + "기록이 없습니다" |
| 앱 첫 설치 (기록 없음) | 빈 캘린더 + EmptyState 안내 |
| 루틴이 삭제된 과거 기록 | 삭제된 루틴은 이름 대신 "(삭제된 루틴)" 표시 |
| 미래 날짜 탭 | 바텀시트 표시하지 않음 (탭 무시) |
| 해당 월에 활성 루틴이 없었던 날 | 0% 달성률 (빈 셀) |
