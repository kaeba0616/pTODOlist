# pTODOlist - PRD (Product Requirements Document)

## 1. 유저 타입

### 단일 유저 (로컬)
- 앱을 설치한 개인 사용자
- 인증/로그인 불필요
- 모든 데이터는 디바이스 로컬에 저장

---

## 2. 문제 정의

매일 반복해야 하는 루틴(운동, 공부, 습관 등)과 그날 추가로 해야 할 일을
한 곳에서 관리하고, 빠뜨린 항목을 자기 전에 알림으로 확인하고 싶다.

---

## 3. 핵심 기능 & 우선순위

### MUST (Phase 1)
- F0: 앱 셸 & 네비게이션
- F1: 카테고리 관리
- F2: 루틴 관리
- F3: 추가 할 일
- F4: 오늘 화면 (홈)
- F5: 일일 기록 & 자정 리셋

### SHOULD (Phase 2)
- F6: 달성률 통계
- F7: 11시 스마트 알림

### COULD (Phase 3)
- F8: 설정 & 데이터 관리

### WON'T
- 서버/클라우드 동기화
- 소셜 기능
- 요일별 루틴 (향후 고려)

---

## 4. 유저 스토리 & Acceptance Criteria

### US-01: 카테고리 관리 (F1)

**As a** 사용자
**I want to** 할 일을 카테고리별로 분류하고 싶다
**So that** 루틴과 할 일을 체계적으로 관리할 수 있다

**Acceptance Criteria:**
- [ ] AC-01-1: 기본 카테고리 5개가 앱 첫 실행 시 자동 생성된다 (운동, 공부, 업무, 생활, 기타)
- [ ] AC-01-2: 새 카테고리를 추가할 수 있다 (이름, 색상 필수)
- [ ] AC-01-3: 기존 카테고리의 이름과 색상을 수정할 수 있다
- [ ] AC-01-4: 카테고리를 삭제할 수 있다
- [ ] AC-01-5: 카테고리 삭제 시 해당 카테고리의 루틴/할 일은 "기타"로 재할당된다
- [ ] AC-01-6: "기타" 카테고리는 삭제할 수 없다

---

### US-02: 루틴 관리 (F2)

**As a** 사용자
**I want to** 매일 반복하는 할 일을 루틴으로 등록하고 싶다
**So that** 매일 체크하면서 빠뜨리지 않을 수 있다

**Acceptance Criteria:**
- [ ] AC-02-1: 새 루틴을 추가할 수 있다 (제목 필수, 카테고리 선택)
- [ ] AC-02-2: 루틴의 제목과 카테고리를 수정할 수 있다
- [ ] AC-02-3: 루틴을 삭제할 수 있다 (확인 다이얼로그)
- [ ] AC-02-4: 루틴을 비활성화할 수 있다 (삭제하지 않고 숨기기)
- [ ] AC-02-5: 루틴의 순서를 드래그로 변경할 수 있다
- [ ] AC-02-6: 비활성화된 루틴은 오늘 화면에 표시되지 않는다

---

### US-03: 추가 할 일 관리 (F3)

**As a** 사용자
**I want to** 오늘 추가로 해야 할 일회성 할 일을 등록하고 싶다
**So that** 루틴 외에 해야 할 일도 한 화면에서 관리할 수 있다

**Acceptance Criteria:**
- [ ] AC-03-1: 새 할 일을 추가할 수 있다 (제목 필수, 카테고리 선택, 날짜 기본값=오늘)
- [ ] AC-03-2: 할 일의 완료 상태를 토글할 수 있다
- [ ] AC-03-3: 완료된 할 일은 섹션 하단으로 이동하고 스타일이 변경된다 (취소선 등)
- [ ] AC-03-4: 할 일을 삭제할 수 있다 (스와이프)
- [ ] AC-03-5: 할 일은 자정에 자동 초기화되지 않는다 (루틴과 다름)

---

### US-04: 오늘 화면 (F4)

**As a** 사용자
**I want to** 앱을 열면 오늘의 루틴과 할 일을 한 눈에 보고 싶다
**So that** 바로 체크하며 진행할 수 있다

**Acceptance Criteria:**
- [ ] AC-04-1: 상단에 오늘 날짜가 표시된다
- [ ] AC-04-2: 오늘의 전체 달성률이 원형 프로그레스로 표시된다
- [ ] AC-04-3: "오늘의 루틴" 섹션에 활성 루틴이 체크박스와 함께 나열된다
- [ ] AC-04-4: "추가 할 일" 섹션에 오늘 날짜의 할 일이 나열된다
- [ ] AC-04-5: 체크박스를 탭하면 완료/미완료가 토글된다
- [ ] AC-04-6: FAB(+) 버튼으로 루틴 또는 할 일을 추가할 수 있다
- [ ] AC-04-7: 루틴/할 일은 카테고리별로 그룹핑된다

---

### US-05: 자정 자동 초기화 (F5)

**As a** 사용자
**I want to** 매일 자정에 루틴의 체크 상태가 자동으로 리셋되길 원한다
**So that** 매일 아침 깨끗한 상태에서 새로 시작할 수 있다

**Acceptance Criteria:**
- [ ] AC-05-1: 자정이 지나면 모든 루틴의 완료 상태가 미완료로 초기화된다
- [ ] AC-05-2: 이전 날의 완료 기록은 보존된다 (통계용)
- [ ] AC-05-3: 앱이 포그라운드에서 자정을 넘기면 즉시 초기화된다
- [ ] AC-05-4: 앱이 백그라운드에서 복귀 시 날짜가 바뀌었으면 초기화된다
- [ ] AC-05-5: 며칠 만에 앱을 열어도 오늘 날짜의 새 기록이 생성된다
- [ ] AC-05-6: 추가 할 일은 자정에 초기화되지 않는다

---

### US-06: 달성률 통계 (F6)

**As a** 사용자
**I want to** 내 루틴 달성률을 일별/주별/월별로 보고 싶다
**So that** 꾸준히 실천하고 있는지 시각적으로 확인할 수 있다

**Acceptance Criteria:**
- [ ] AC-06-1: 통계 탭에서 일별 달성률을 막대 차트로 볼 수 있다 (최근 7일)
- [ ] AC-06-2: 주별 평균 달성률을 볼 수 있다 (최근 4주)
- [ ] AC-06-3: 월별 평균 달성률을 볼 수 있다 (최근 6개월)
- [ ] AC-06-4: 기간 선택 탭(일/주/월)으로 전환할 수 있다
- [ ] AC-06-5: 카테고리별 달성률 분석을 볼 수 있다

---

### US-07: 11시 스마트 알림 (F7)

**As a** 사용자
**I want to** 밤 11시에 아직 완료하지 못한 일이 있으면 알림을 받고 싶다
**So that** 자기 전에 빠뜨린 일을 마무리할 수 있다

**Acceptance Criteria:**
- [ ] AC-07-1: 밤 11시에 미완료 항목이 있으면 푸시 알림이 발생한다
- [ ] AC-07-2: 알림 내용에 미완료 항목 수와 일부 항목명이 표시된다
- [ ] AC-07-3: 모든 항목이 완료되었으면 알림이 발생하지 않는다 (스마트 무음)
- [ ] AC-07-4: 설정에서 알림을 끌 수 있다
- [ ] AC-07-5: 알림을 탭하면 오늘 화면으로 이동한다

---

### US-08: 설정 & 데이터 관리 (F8)

**As a** 사용자
**I want to** 알림 설정과 데이터 보관 기간을 관리하고 싶다
**So that** 앱을 내 취향에 맞게 사용할 수 있다

**Acceptance Criteria:**
- [ ] AC-08-1: 알림 ON/OFF 토글이 있다
- [ ] AC-08-2: 데이터 보관 기간을 선택할 수 있다 (3개월/6개월/1년/무제한)
- [ ] AC-08-3: 보관 기간이 지난 DailyRecord는 앱 실행 시 자동 삭제된다
- [ ] AC-08-4: 테마를 변경할 수 있다 (라이트/다크/시스템)
- [ ] AC-08-5: 앱 버전 정보가 표시된다
