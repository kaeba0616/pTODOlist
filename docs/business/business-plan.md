# pTODOlist - 사업계획서

## 1. 프로젝트 개요

### 프로젝트명
pTODOlist (Personal TODO List)

### 한 줄 소개
매일 반복하는 루틴과 추가 할 일의 완료 여부를 추적하는 로컬 전용 모바일 앱

### 비전
누구나 쉽게 매일의 루틴을 관리하고, 꾸준한 습관을 만들어갈 수 있는 가장 심플한 도구

---

## 2. 문제 정의

### 페인 포인트
1. **루틴 관리의 어려움**: 매일 해야 할 일을 기억에만 의존하면 빠뜨리기 쉬움
2. **기존 앱의 복잡성**: Todoist, TickTick 등은 기능이 많아 습관 관리 용도로 과함
3. **수동 리셋 번거로움**: 매일 아침 어제 완료 표시를 일일이 해제해야 함
4. **알림 피로**: 고정 시간 알림은 이미 다 한 날에도 울려서 피로감 유발
5. **프라이버시 우려**: 서버에 일정을 올리고 싶지 않은 사용자 존재

### 기회
- 습관 트래커 시장 지속 성장 (글로벌 self-improvement 시장)
- "미니멀 프로덕티비티" 트렌드: 적은 기능 + 핵심에 집중
- 로컬 전용 = 서버 비용 없음, 유지보수 부담 최소

---

## 3. 타겟 시장

### 주요 타겟
- **연령**: 20~40대
- **특징**: 자기 관리에 관심 있으나, 복잡한 앱에 질린 사용자
- **행동 패턴**: 매일 아침/저녁 루틴이 있거나 만들고 싶은 사람

### 페르소나
> **김민수 (28세, 직장인)**
> 매일 아침 운동, 독서, 영어 공부를 하고 싶지만 꾸준히 못 한다.
> 기존 TODO 앱은 업무용으로 쓰고 있어서, 개인 루틴용 심플한 앱을 원한다.
> 자기 전에 "오늘 뭘 빠뜨렸는지" 알려주면 좋겠다.

## 4. 핵심 기능 (MoSCoW)

### MUST (Phase 1)
- 매일 루틴 등록/수정/삭제/체크
- 추가 할 일 등록/완료/삭제
- 카테고리 분류 (운동, 공부, 업무, 생활, 기타)
- 자정 자동 초기화 (DailyRecord 패턴)
- 오늘의 달성률 표시

### SHOULD (Phase 2)
- 달성률 통계 (일별/주별/월별 차트)
- 11시 스마트 알림 (미완료 항목만, 전부 완료면 무음)

### COULD (Phase 3)
- 데이터 보관기간 설정 (3/6/12개월)
- 다크 모드
- 데이터 내보내기/백업

### WON'T (현재 범위 밖)
- 서버/클라우드 동기화
- 소셜 기능 (친구, 공유)
- 요일별 다른 루틴
- 위젯

---

## 5. 기술 전략

### 아키텍처
- **Frontend**: Flutter (iOS + Android 단일 코드베이스)
- **상태관리**: Riverpod
- **로컬 DB**: Hive
- **Backend**: 없음 (로컬 전용)

### 기술적 차별점
- 서버 비용 $0
- 오프라인 100% 동작
- 앱 용량 최소화 (외부 서비스 의존 없음)
- 개인 데이터가 디바이스 밖으로 나가지 않음

---

## 6. 수익 모델

### 현재: 개인 프로젝트
- 무료 배포, 수익화 없음

### 향후 가능성
- 프리미엄 테마/아이콘 팩 (일회성 구매)
- 고급 통계 (연간 리포트, 카테고리 분석)
- 위젯 기능 (프리미엄)
