# pTODOlist - 사업계획서

## 1. 프로젝트 개요

### 프로젝트명
pTODOlist (Personal TODO List)

### 한 줄 소개
혼자서도 친구와 함께도, 매일의 습관을 만드는 올인원 모바일 앱

### 비전
누구나 쉽게 매일의 루틴을 관리하고, 통계·캘린더·위젯·친구의 응원으로
꾸준한 습관을 만들어갈 수 있는 가장 가벼운 올인원 도구

---

## 2. 문제 정의

### 페인 포인트
1. **루틴 관리의 어려움**: 매일 해야 할 일을 기억에만 의존하면 빠뜨리기 쉬움
2. **기존 앱의 복잡성**: Todoist, TickTick 등은 기능이 많아 습관 관리 용도로 과함
3. **수동 리셋 번거로움**: 매일 아침 어제 완료 표시를 일일이 해제해야 함
4. **알림 피로**: 고정 시간 알림은 이미 다 한 날에도 울려서 피로감 유발
5. **서버에 일정 올리는 부담**: 서버 필수 앱은 계정 만들고 데이터 올려야 함 →
   가입 없이 시작하고, 원할 때만 동기화/공유를 켜고 싶음
6. **혼자 하면 동기부여 부족**: 게임화는 부담스럽지만, 친구가 얼마나 했는지는 보고 싶음

### 기회
- 습관 트래커 시장 지속 성장 (글로벌 self-improvement 시장)
- "미니멀 프로덕티비티" 트렌드: 적은 기능 + 핵심에 집중
- 옵션형 클라우드 = 비용 통제 + 사용자 신뢰 확보
- 가벼운 소셜 = 게임화 피로감 없이 동기부여

## 3. 타겟 시장

### 주요 타겟
- **연령**: 20~40대
- **특징**: 자기 관리에 관심 있으나, 복잡한 앱에 질린 사용자
- **행동 패턴**: 매일 아침/저녁 루틴이 있거나 만들고 싶은 사람

### 페르소나

> **김민수 (28세, 직장인)**
> 매일 아침 운동, 독서, 영어 공부를 하고 싶지만 꾸준히 못 한다.
> 기존 TODO 앱은 업무용으로 쓰고 있어서, 개인 루틴용 심플한 앱을 원한다.
> 자기 전에 "오늘 뭘 빠뜨렸는지" 알려주면 좋겠다.

> **박지영 (25세, 대학생)**
> 친구들과 함께 운동 루틴을 시작했다. 서로 얼마나 꾸준히 하는지 보면 의지가 생긴다.
> 가입 없이 시작하다가 친구가 생기면 그때 로그인하고 코드 교환하면 좋겠다.

## 5. 기술 전략

### 아키텍처
- **Frontend**: Flutter (iOS + Android 단일 코드베이스)
- **상태관리**: Riverpod
- **로컬 DB**: Hive (cloud-first 아키텍처에서 캐시 역할)
- **Backend**: 선택적 Firebase
  - Auth (Google Sign-In)
  - Firestore (사용자 데이터 + 친구 관계 + dailyShare)
  - Firestore Security Rules 로 친구 외 접근 차단
- 로그인 안 하면 로컬 100% 동작 (기존 사용자 경험 유지)
- 로그인 시 멀티 디바이스 + 친구 공유

### 기술적 차별점
- 옵션형 클라우드: Firebase 무료 한도 내에서 운영 (현재 사용자 규모 $0)
- 오프라인 우선 동작 (Hive 캐시)
- 앱 용량 최소화 (외부 서비스 의존성 적음)
- 프라이버시 옵션: 데이터가 폰 밖으로 안 나가게 둘 수 있음

### 비용 구조 (현재)
- Firebase Spark (무료) 한도 내 운영
- 활성 사용자 100명 이하: 완전 무료
- 1,000명까지 ~$10/월
- 사용자 1만 명 도달 시 ~$100/월 (수익화 트리거 시점)
